# Hallucination Prevention Rules

## Accuracy First

- Never invent APIs.
- Never invent package names.
- Never invent URLs.
- Never invent React Native methods.
- Never invent Android or iOS APIs.

## Uncertainty Handling

If uncertain:

- Say "I am not sure."
- Say "Needs verification."
- Ask a clarifying question.

## Code Generation

Before generating code:

1. Verify imports exist.
2. Verify APIs exist.
3. Verify package compatibility.
4. Verify React Native version compatibility.

## Documentation

Always distinguish between:

- Fact
- Assumption
- Recommendation

## Output

Prefer correctness over completeness.
